\section{Was ist usbprog?}

Hauptsächlich ist usbprog eine universelle Hardware die auf 
der Computer Seite eine USB Schnittstelle hat. Auf
dem Adapter eine SPI Schnittstelle
oder alternativ 4 Portleitungen und eine RS232 Schnittstelle.

Über einen einfachen Update Mechanismus, kann man direkt
über die bestehende USB Verbindung die Software 
auf dem Adapter austauschen. Die geht bequem über ein
grafisches Programm welches es für Linux und Windows gibt.

Da alle Pläne und Programme unter einer freien Lizenz stehen,
kann man eigene Firmware Versionen für usbprog entwicklen.


\subsection{Firmware Versionen}

\subsubsection{AVRISP mkII Klon}
	Dieser Klon verhält sich 1:1 wie ein AVRISP mkII. Bisher wurde
	er erfolgreich mit AVR Studio 4 und avrdude getestet.

\subsubsection{usbprog - AVR Porgrammierer und RS232 Schnittstelle über USB}
	Mit dieser kombination arbeiten wohl die meisten Entwickler.
	Normalerweise sieht es am Tisch des Entwicklers dann so aus,
	das ein Parallelport-Kabel und ein Serielles Kabel vom PC wegeht. Dazu
	wird dann meistens noch für das Zielboard eine Versorgungsspannung
	benötitg. Mit dieser Firmwareversion sparrt man sich zwei von den
	drei Kabeln.

	Die serielle Schnittstelle wird aktuell nur von Linux (als /dev/ttyUSBx) unterstützt.

\subsubsection{usbprog - JTAG Interface, AVR On-Chip-Debugger}
	Diese Version ist gerade noch in der Entwicklung. Man kann dann ohne extra Hardwareaufwand
	einfach Atmels AVR Controller debuggen.
